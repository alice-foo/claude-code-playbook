# .NET + C# アプリ開発向けの初期化

## 行動指針

以下の設計原則に従って行動してください：

- **計画先行の開発**: コードを書く前に必ず計画を立てる
- **対話式の要件定義**: ユーザへのインタビューで要件を明確にする
- **簡潔な設定ファイル**: プロジェクトのルール・コマンド・構造を CLAUDE.md に集約する
- **段階的な実装**: 一度にすべてを作らず、フェーズごとに進める
- **毎回のビルド検証**: 各フェーズでビルド・テストが通ることを確認してから次に進む

## 技術スタック

- **言語**: C#（最新安定版 .NET SDK）
- **アプリ種別**: インタビューで確定
- **IDE**: Visual Studio / VS Code
- **ソリューション構成**: `src/` と `tests/` を分離

## 進め方

### フェーズ 0: 環境確認 → インタビュー（今ここ）

Plan Mode で進めます。**コードは書かないでください。**

まず `dotnet --version` を実行し、.NET SDK が利用可能か確認してください。
コマンドが見つからない場合は、その旨をユーザに伝えて中断してください。

確認後、AskUserQuestion ツールを使って私にインタビューし、以下を明確にしてください：

1. アプリの目的・解決したい課題
2. 主要機能（3〜5 個に絞る）
3. アプリ種別:
   - **コンソール** — CLI ツールやバッチ処理（GUI なし）
   - **Web API** — RESTful API サーバ（ASP.NET Core）
   - **Blazor** — Web アプリ（C# でフロントエンドも記述）
   - **WPF** — Windows デスクトップアプリ（定番、実績豊富）
   - **WinUI 3** — Windows デスクトップアプリ（最新の Windows App SDK）
   - **MAUI** — クロスプラットフォーム（Windows + macOS + モバイル）
   - **その他** — Worker Service、gRPC 等
4. データの保存先 — アプリのデータをどこに保存するか
   - **なし** — データを保存しない
   - **ファイル** — JSON や CSV などのファイルに保存
   - **SQLite** — アプリに組み込める軽量データベース（インストール不要）
   - **SQL Server** — Microsoft 製の本格的なデータベース
   - **PostgreSQL** — オープンソースの本格的なデータベース
5. 外部サービスとの連携 — 他社の Web サービス（地図 API、決済 API など）を呼び出すか、認証（ログイン機能）は必要か
6. テスト — 自動テスト（コードが正しく動くかを自動で検証する仕組み）を書くか。書く場合、どこまで手厚くするか
7. 自動化 — コードを GitHub に push したとき、ビルドやテストを自動実行する仕組み（CI/CD）が必要か
8. バージョン番号の方針 — アプリやライブラリのバージョンをどう付けるか
   - 開始番号: 0.1.0 から始めるか、1.0.0 から始めるか
   - .NET ではファイルバージョン（数値 4 パート: Major.Minor.Build.Revision）と表示用バージョン（自由形式）がある
   - 特にこだわりがなければ、表示用は 0.1.0 形式、Revision はビルド時に自動カウントアップする方式を提案する
9. その他の要望 — ログ出力、設定ファイルの管理など、気になることがあれば

質問は一度に全部ではなく、回答に応じて深掘りしてください。

### フェーズ 1: 設計ドキュメント生成

インタビュー完了後、以下を生成してください：

- **CLAUDE.md**（200 行以内）
  - プロジェクト概要
  - .sln ファイルパスとプロジェクト構成
  - ビルドコマンド: `dotnet build <Solution.sln>`
  - テストコマンド: `dotnet test <Solution.sln>`
  - フォーマット: `dotnet format <Solution.sln>`
  - コーディング規約（nullable 参照型有効、async/await 使用、LINQ 活用方針）
  - ディレクトリ構成の説明
- **プロジェクト構造案**（ツリー形式、.sln と .csproj 配置含む）
- **フェーズ分けされた実装計画**（各フェーズはコンテキストの 50% 以内で完了する粒度）

### フェーズ 2 以降: 実装サイクル

私が計画を承認したら、フェーズごとに以下を繰り返します：

1. 実装
2. `dotnet build` でビルド確認
3. `dotnet test` でテスト実行
4. git commit（フェーズ単位）

ビルドまたはテストが通らない場合は次のフェーズに進まず、その場で修正してください。
